Pomodoro Timer

This project was a challenge to myself to complete within a weekend. I realized I needed to practice Java and just start coding more often! The title links to this project's repository on my GitHub where you can see more.

This timer is meant to be used for productivity and follows the pomodoro technique. The general concept of the pomodoro technique is that with frequent short breaks you can be more productive and will retain more information. Traditionally each pomodoro, the working period, lasts 25 minutes followed by a 5 minute break. Every fourth pomodoro you take a longer 30 minute break before continuing along the same pattern. I went ahead and made my timer take in user input for custom work and break durations, each fourth pomodoro simply adds these two values for the long break.

To create this project I used JavaFX 16, SceneBuilder, and both Java's TimerTask and Timer classes. One thing that I orginally did not realize when beginning this project was that Timer objects run on separate threads. This certainly was very confusing to me when I tested my code for the first time and the timer was decreasing way faster than it should have. My solution to this problem, after realizing the cause, was to have one timer task track the the working period and then start into the break period. I can admit that this is not the most optimal solution, but now I know what to expect in the future!

Inventory Manager

This was the final project for my Software I course. We had to use JavaFX and Java and were allowed to use SceneBuilder to create the GUI. This was my first large project and was a very rewarding experience in many ways. The title links to this project's repository on GitHub where you can see more.

The situation we were given was to create an inventory management system for a small manufacturing company that was using spreadsheets for inventory management. The system had to organize parts and products, along with their associated parts. Specific part or product information could be modifed, be deleted, and of course simply be viewed.

This project was the project that really solidified my understanding of Object Oriented Programming. Before this project the concepts of inheritance and abstract classes always confused me. Clarification came from seeing how In-House and Outsourced objects could be instatiated, but a plain old Part object could not be instatianted. This helped me realize that subclasses of abstract classes force typing within that super class. Ultimately, this project gave me a concrete foundation for Object Oriented Programming and its associated concepts.

/**
* Handles the event of the save button being clicked.
* Adds a new product to the inventory with user specified data.
* 
* Gives user a warning message for bounds errors such as:
* Minimum stock larger than maximum stock
* Current stock larger than maximum stock
* Current stock less than minimum stock
* 
* Gives user a warning message for non-numerical characters in any
* fields requiring numerical values
* 
* @param event
* @throws IOException 
*/
public void saveButtonPushed(ActionEvent event) throws IOException {
    try {
        int ID = getProductIDCount();
        String name = productName.getText();
        int stock = Integer.parseInt(productStock.getText());
        double price = Double.parseDouble(productPrice.getText());
        int max = Integer.parseInt(productMax.getText());
        int min = Integer.parseInt(productMin.getText());
        if (min > max) {
            Alert bounds = new Alert(Alert.AlertType.WARNING);
            bounds.setTitle("Bounds Error");
            bounds.setHeaderText("Mininum stock must be less than maximum stock.");
            bounds.setContentText("Please fix this error!");
            bounds.showAndWait();
        } else if (stock < min) {
            Alert bounds = new Alert(Alert.AlertType.WARNING);
            bounds.setTitle("Bounds Error");
            bounds.setHeaderText("Product stock must exceed minimum stock.");
            bounds.setContentText("Please fix this error!");
            bounds.showAndWait();
        } else if (stock > max) {
            Alert bounds = new Alert(Alert.AlertType.WARNING);
            bounds.setTitle("Bounds Error");
            bounds.setHeaderText("Product stock must NOT exceed maximum stock.");
            bounds.setContentText("Please fix this error!");
            bounds.showAndWait();
        } else {
            Product product = new Product(ID, name, price, stock, max, min);
            for (Part part : productAssociatedParts) {
                product.addAssociatedPart(part);
            }
            addProduct(product);
            Parent mainParent = FXMLLoader.load(getClass().getResource("/view/Main.fxml"));
            Scene MainScene = new Scene(mainParent);
            Stage window = (Stage)((Node)event.getSource()).getScene().getWindow();
            window.setScene(MainScene);
            window.show();
        }
    } catch (NumberFormatException exception) {
        Alert format = new Alert(Alert.AlertType.WARNING);
        format.setTitle("Format Error");
        format.setHeaderText("Please ensure that input type is appropriate!");
        format.setContentText("Please fix this error!");
        format.showAndWait();
    }
}

Scrape LKQ

This project was made in Python using Selenium for my current job. In the car rim business there is one company, LKQ, that sets the rules. Due to this we use their part numbers for as much consistency as possible. The title links to this project's repository on GitHub where you can see more.

Our inventory gets fed into eBay listings and into listings on our Website. The product details are usually taken off of LKQ's website manually and put into our database for accuracte product specifics, which is important for fitment. This project scrapes all needed rim information from LKQ's website and organizes it so I can send it to our IT team.

Although I have used Selenium in the past, I feel that this time I used it more efficiently. This combined with the Object Oriented Programming concepts I have been learning made this project a breeze. Each part number search creates a new rim object and as information is scraped from the website the rim object's attributes are set. Each valid rim object is added to an array which is then iterated through in order to write each rim and rim specifics to an output csv file.

def stripInfo(browser: webdriver, rim: Rim):
    """
    Modifies a rim object to add necessary data
        Keyword Arguments:
            browser (webdriver): current browser instance
            rim (Rim): rim object being modified
    """
    def returnMatGridText(browser: webdriver, css_selector: str) -> List[str]:
        """
        Selects and enters hollander into search field
            Keyword Arguments:
                browser (webdriver): current browser instance
                css_selector (str): css selector for specific element type
            Returns:
                outp (list): list of data elements
        """
        elements = wait.until(EC.visibility_of_element_located((By.CSS_SELECTOR, cssSelector)))
        elements = browser.find_elements_by_css_selector(css_selector)
        outp = []
        for element in elements:
            outp.append(element.text.strip())
        return outp
    
    # Strip Shown Details
    wait = WebDriverWait(browser, 5)
    try:
        partDescriptionPath = r'//*[@id="search-results-container"]/div/div/app-product-card/div/section/div[3]/span[2]'
        partDescription = wait.until(EC.visibility_of_element_located((By.XPATH, partDescriptionPath)))
        text = partDescription.text
        temp = text.split('\n')
        info = []
        for i in range(len(temp)):
            info.append(temp[i][2:])

        try:

            partDescription = wait.until(EC.visibility_of_element_located((By.XPATH, partDescriptionPath)))
            text = partDescription.text
            temp = text.split('\n')
            info = []
            for i in range(len(temp)):
                info.append(temp[i][2:])
        except:
            print(f"No results for {rim.sku}")
    
    # Info list will be passed in
    # Validation done here!
    info = validateDescription(info)

    rim.finish = str(info[0])
    rim.bp = str(info[1])
    rim.lugs = str(info[2])
    rim.style = str(info[3])

    # Strip Image    
    # Tries to access the other information by clicking image first

    try:
        imagePath = r'//*[@id="search-results-container"]/div/div/app-product-card/div/section/div[2]/img'
        image = browser.find_element_by_xpath(imagePath)

    try:
        imageSRC = image.get_attribute('src')

        print(f'Failed to retrieve extra info for {rim.sku}')

C2C Check

This script was created due to my frustration of having a large excel window open in my already cluttered virtual workspace. Having to search through the excel sheet is as simple as pressing CTRL + F, but why do that when I can make it smaller and just type in my search? This title does not redirect to a GitHub repository as the Code Snippet box contains the entire program code!

On our website and eBay account we list the inventory of a LKQ subsidiary, Coast to Coast, as available. This means that I often have to search through their inventory when fulfilling orders as well as ours.

The script simply uses python's csv module to read in the information stored in the inventory csv. The script searches for partial matches so that I can view all of the avalable color options. As the file's rows are iterated through, an output string is built with numbered results, part number, and quantity. The numbered results was something I originally did so that I could easily incorporate automatic drop ship emails being sent if I selected a search result's number. This idea turned out to be unnecessary as most searches do not result in an immediate drop ship being made.

Code Snippet

                    
import csv

selection = input("Enter a hollander to search for!\nOr quit to quit the program.\n")
while selection.lower() != "quit":
    output = ""
    with open(r"C:\Users\danny\Work\C2C\C2C.csv", newline = '') as csvfile:
        reader = csv.reader(csvfile, delimiter=',', quotechar='|', quoting=csv.QUOTE_MINIMAL)
        count = 1
        for row in reader:
            if selection in row[1]:
output += str(count) + ":\t" + row[1] + "\t" + row[4] + "\n"
count += 1
        if output == "":
            print("\nNo results!\n")
        else:
            print(output)
    selection = input("Enter a hollander to search for!\nOr quit to quit the program.\n\n")
